# Adopt In-Memory Caching Layer for Package Metadata and HTTP Responses: Site Packages Metadata

These rules are ALWAYS ACTIVE for all package management operations, HTTP client interactions, metadata resolution workflows, and site-packages scanning within the uv package manager codebase.

### Rules

- **R-CACHE-001** MUST: Site-packages metadata and installed package information MUST be cached to avoid repeated filesystem scans during dependency resolution.
- **R-CACHE-002** MUST: HTTP client responses from package registries and indices MUST be cached in-memory to reduce network latency and repeated requests.
- **R-CACHE-003** MUST: Authentication credentials and tokens MUST be cached to reduce authentication overhead, except when security-sensitive operations require fresh validation (EXC-002).
- **R-CACHE-004** MUST: Parsed HTML and JSON metadata from package indices MUST be cached to avoid re-parsing identical responses.
- **R-CACHE-005** MUST: Cache implementations MUST use thread-safe concurrent data structures (e.g., once_map, RwLock) to prevent data corruption in concurrent access scenarios.
- **R-CACHE-006** MUST: HTTP caching MUST comply with RFC 7234 to respect upstream cache-control directives.
- **R-CACHE-007** MUST: Cache keys MUST be derived from cryptographic digests to ensure uniqueness and integrity.
- **R-CACHE-008** SHOULD: Cache size limits and LRU eviction policies SHOULD be implemented to prevent memory exhaustion in large dependency tree operations.
- **R-CACHE-009** SHOULD: CLI flags for cache control (--no-cache, --refresh) SHOULD be provided to allow users to bypass caching when needed (EXC-001).
- **R-CACHE-010** SHOULD: Cache statistics and monitoring SHOULD be implemented to track hit rates and identify optimization opportunities.

### Verify

```bash
# Count cache-related patterns across codebase
grep -r 'cache' crates/uv-*/src/ | grep -E '(Cache|cache_|cached)' | wc -l

# Find all cache-related source files
find crates/ -name '*cache*.rs' -type f

# Verify once_map pattern usage for concurrent caching
grep -r 'once_map\|OnceMap' crates/ --include='*.rs'

# Run cache-related tests
cargo test --package uv-client --test '*cache*' 2>&1 | grep -E '(test.*ok|passed)'
```

**Accept when:**
- Cache-related files and modules are present in the codebase (httpcache, auth cache, once_map)
- Grep searches for cache patterns return significant matches across multiple crates
- Cache-related tests exist and pass successfully
- Performance benchmarks show reduced network requests and improved execution time compared to no-cache baseline
- Thread-safety of cache implementations is verified by static analysis tools
- Integration tests validate cache behavior with various network conditions

<enforcement>
Clause Code MUST NOT skip or defer verification of cache implementation patterns, thread-safety guarantees, and RFC 7234 compliance. All new data access patterns in performance-critical paths MUST include caching strategy review. Cache hit rate regressions MUST trigger CI failures and require architecture review.
</enforcement>
